# 系統混亂診斷報告 (System Chaos Diagnosis Report)

**執行專員：** Chaos Surveyor (系統混亂測量師)
**狀態：** Draft for Review

> 「在動手談品味之前，先看看你的底子有多爛。」

## 1. 現有架構觀測 (System Topology Observation)

根據客戶提供的資訊與初步 API 觀測，幻影矩陣目前的 AI 對話流呈現以下混亂拓撲：

- **檢索層 (Retrieval Layer)**：採用標準的 Cosine Similarity 進行 RAG 檢索。由於向量資料庫（Vector DB）中存放的是靜態的「人設台詞包」，導致檢索結果高度重複。
- **上下文視窗 (Context Window)**：為了維持射擊遊戲的高並發效能，客戶限縮了緩存量，導致 Agent 在長對局中容易遺忘前 5 分鐘的交火細節。
- **延遲瓶頸**：模型回應時間（TTFT）與遊戲戰鬥幀數（FPS）缺乏同步機制，造成「人已經被打死了，AI 才剛回完嗆聲」的尷尬延時。

## 2. 三大核心架構痛點 (Identified Pain Points)

### 痛點 A：靜態檢索导致的「罐頭感」
目前的 RAG 邏輯僅依據「玩家說了什麼」來檢索，完全忽略了遊戲內的「即時局勢數據」（如：玩家血量、子彈餘量、戰場位置）。
- **後果**：無論玩家是滿血衝鋒還是殘血躲避，AI 抽到的台詞權重都一樣，缺乏「高端玩家」的動態應對感。

### 痛點 B：Embedding 維度坍塌
現有的 Embedding 模型對「遊戲黑話」與「挑釁語氣」的向量捕捉不精準，導致「嘲諷」與「問候」在向量空間中的距離過近。
- **後果**：AI 經常在該嘲諷的時候顯得過於客氣，喪失了遊戲應有的張力。

### 痛點 C：缺乏邊界防禦邏輯
目前的系統直接將 Player Input 丟給 LLM，缺乏前置的惡意輸入過濾器。
- **後果**：Agent 容易被玩家透過「假設性情境」繞過人設，直接曝露底層 Prompt。

## 3. 下階段建議

- **對底層雕刻家建議**：必須在 Embedding 空間中手動拉開「嘲諷度」與「專業度」的維度間距。
- **對極限環境調校師建議**：需針對「同步延遲」進行施壓測試，找出不破壞 Vibe 的最高容忍延時（ms）。
