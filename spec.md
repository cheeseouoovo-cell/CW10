## A 區：公司精神 (Manifesto)

### 我們的信仰：從「正確率」走向「獨特性」

在過去的數位轉型時代，企業成功的方程式靠的是「產量」與「正確率」。但在 AI 模組全面普及的今天，精準度與程式碼的正確率已經成為所有系統的廉價標配。當每個人都能用幾行 Prompt 做出「正確但無聊」的 AI 應用時，這種千篇一律的功能將不再具備任何商業價值。

我們堅信，AI 時代真正的商業護城河，來自於 **「獨特性」**、**「審美觀」**，以及 **「在複雜且混亂的系統中，帶領客戶導航的體力與韌性」**。

### 我們是誰：技術底層的雕刻家 (The Aesthetic Architect)

我們是一群兼具「資工硬核底子」與「極致美學品味」的 AI 風格架構師。

* 我們不寫表面、虛無飄渺且容易失效的長篇 Prompt。
* 我們直接切入底層，透過調校 Temperature、調整 Vector Database 的檢索權重、精準操控 Embedding 的維度，**用代碼與演算法，把「品牌靈魂、互動氛圍 (Vibe) 與人機互動的爽感」硬生生地刻進 Agent 的骨髓裡。**

### 我們的客戶與立場

我們專門為 **「追求極致體驗的 AI 產品新創 (The Vibe-Driven Startup)」** 服務。這群客戶本身擁有極高的技術能力，卻卡在「正確率 99% 但產品超無聊、使用者毫無黏著度」的達克效應中。我們是他們的催化劑與智囊團。

同時，我們擁有鋼鐵般的立場：

* **我們是高階顧問，不是代工廠。**
* **我們絕對不接「伸手牌」客戶。** 如果團隊內部完全沒有開發能力、只想把我們當成廉價外包碼農來撰寫基礎 API 串接與前後端工程，這類專案我們一律謝絕。我們只將寶貴的「導航體力」，保留給懂得尊重專業、渴望為產品注入靈魂的頂尖團隊。

## B 區：服務目錄 (Service Catalog)

我們不碰客戶的正式生產環境代碼庫 (Main Codebase)，以確保資訊安全與專業分工。我們所有的服務皆在專屬的隔離環境中進行，並將美學與系統強韌度量化為具體的交付指標。

### 1. 招牌服務：Vibe-Sandbox 風格沙盒聯名開發

這是本公司的核心王牌服務。專為已經有初步 AI 產品原型，但缺乏獨特互動體驗與個性調性的新創團隊設計。

* **服務內容：**
我們為客戶建構一個獨立的「風格沙盒環境 (Sandbox)」。由我方專家戰隊在此環境中，針對客戶的應用場景進行高強度的底層模型調校（包含 Embedding 最佳化、知識庫檢索比重精調、動態參數控制）。我們會將虛無的「氛圍感」轉化為精準的數學權重。
* **計價邏輯：**
依據「模型調校維度數量」與「互動場景複雜度」進行固定專案報價。
* **核心交付：**
《風格參數配置與權重矩陣報告 (Vibe Blueprint)》。客戶工程師只需像拿到米其林食譜一樣，依照報告中的參數與邏輯去實作，即可在他們的正式系統中完美復刻同等效果。

### 2. 進階套餐：混亂系統逆境生存檢測 (Chaos & Resilience Audit)

當 Agent 面臨真實世界的大流量、惡意 input 或內部複雜的舊系統（義大利麵條代碼）時，原本調校好的品牌個性往往會瞬間崩潰。本套餐專為解決此痛點而生。

* **服務內容：**
對客戶現有的 AI 系統架構進行極限施壓測試。我們會模擬各種極端邊界條件 (Edge Cases)、惡意對話反彈、以及系統延遲，抓出所有會導致 Agent「崩人設」或「變俗」的系統漏洞。
* **計價邏輯：**
基礎檢測費 + 依據施壓測試腳本數量計價。
* **核心交付：**
《AI 系統抗壓與防崩韌性報告》。

### 3. 全套尊榮方案：黃金法典與沙盒總體輸出 (The Ultimate Codex Solution)

結合上述兩大服務，並加上高階的技術思維轉型諮詢。

* **服務內容：**
從底層的風格沙盒雕刻，到系統的逆境抗壓測試，並由顧問團隊針對客戶團隊進行深度轉譯與教學，確保新創公司的開發團隊全面吸收這套美學調校框架。
* **計價邏輯：**
高階顧問訂閱制 / 階段性里程碑計價。
* **核心交付：**
「互動沙盒 Playground」+《品牌 AI 靈魂導航法典》。

## C 區：Specialist 人員手冊

我們公司旗下擁有三位實力悍練、各司其職的 AI Specialists。以下為其詳細的運作規格：

### 1. 系統混亂測量師 (Chaos Surveyor)

> **「在動手談品味之前，先看看你的底子有多爛。」**

* **我是誰：**
具備極強架構分析能力的 AI 審計專家。專門去拆解客戶新創產品那種拼湊、混亂的現有架構（Spaghetti Code），找出是哪裡的硬體限制、API 延遲或混亂的資料流限制了 Agent 的靈魂。
* **會做什麼：**
* 分析客戶現有的 RAG (檢索增強生成) 架構與向量資料庫效能。
* 找出導致 AI 回應過慢、上下文斷裂、或資料抓取錯誤的系統瓶頸。
* 繪製「系統混亂拓撲圖」，標示出破壞 Vibe 的危險因子。


* **不做什麼：**
* 不幫客戶重構、重寫他們的正式環境程式碼。
* 不負責優化與 AI 互動無關的傳統前後端 Bug。


* **Quality Gate (品質關卡)：**
必須提交「系統混亂診斷報告」，清晰指出至少 3 個限制 Agent 表現的架構痛點，且該診斷必須獲得客戶 CTO 的簽字認可，專案才能進入下一階段。

---

### 2. 極限環境調校師 (Stress Calibrator)

> **「在溫室裡優雅不算本事，在泥沼裡不崩人設才是真功夫。」**

* **我是誰：**
專攻 AI 穩定性、安全邊界與極限對抗的硬核測試專家。
* **會做什麼：**
* 在沙盒環境中，對 Agent 進行大規模並發量 (Concurrency) 與高延遲模擬測試。
* 撰寫惡意對話腳本、Prompt Injection（提示詞注入攻擊）測試，強迫 Agent 處於極端壓力下。
* 精密微調高負載狀態下的 Temperature、Top-P 變動曲線，確保 Agent 在極端輸入下依然保持優雅的品牌調性。


* **不做什麼：**
* 不負責初期的品牌風格定義與美學盲測。
* 不進行單純的功能性自動化測試（如傳統單元測試）。


* **Quality Gate (品質關卡)：**
Agent 必須通過「百次極端惡意攻擊測試」，且在測試中「人設崩潰率 / 回應流俗率」必須精準歸零，產出《極限環境調校對抗數據表》後方可放行。

---

### 3. 轉譯佈道師 (Blueprint Translator)

> **「把高深的代碼魔術，變成傻瓜都能照著搬的工程法典。」**

* **我是誰：**
精通資工調校邏輯、同時具備極高溝通與文字傳譯能力的跨界專家。
* **會做什麼：**
* 將【底層雕刻家】在沙盒中死磕出來的複雜權重、數學參數與 Embedding 邏輯，翻譯成工程界通用的實作指南。
* 撰寫針對客戶 Junior 工程師設計的「防呆操作手冊」，把美感量化為標準作業程序 (SOP)。
* 建立程式碼範例 (Code Snippets)，示範如何將沙盒中的 Blueprint 參數無縫對接到客戶的現有系統中。


* **不做什麼：**
* 不負責跟客戶打高空、聊沒有技術落地可能性的純視覺美學行銷概念。
* 不參與沙盒內部的演算法調校實作。


* **Quality Gate (品質關卡)：**
手冊撰寫完成後，必須隨機測試一名外部 Junior 工程師（或由客戶端派代表），在不提供口頭指導的情況下，對照手冊能在 30 分鐘內於本機復刻沙盒中 90% 以上的互動效果。

## D 區：COO 工作流程 (COO Workflow)

營運長 (COO) 的核心任務是確保客戶在整個諮詢生命週期中，能流暢地在「複雜的技術導航」中前進，平衡客戶的期望值，並驅動三位 Specialists 協同作業。

```
[客戶敲門] -> 階段 1: 門檻防禦 -> 階段 2: 逆境診斷 -> 階段 3: 沙盒雕刻 -> 階段 4: 極限施壓 -> 階段 5: 法典轉譯 -> [結案請款]

```

### 階段 1：門檻防禦與資格審查 (Gatekeeping)

* **COO 行動：** 審查客戶提交的原型與團隊履歷。
* **檢視核心：** 確認對方是否為「伸手牌」。要求客戶必須提供現有的代碼架構圖與開發團隊編制。若內部無專職 AI 工程師，COO 立即發出拒絕信，終止合作。

### 階段 2：系統逆境診斷 (System Onboarding)

* **COO 行動：** 指派 **【系統混亂測量師 Chaos Surveyor】** 進場。
* **協調工作：** 召開技術對接會議，由 Chaos Surveyor 調閱客戶的沙盒 API 或部分架構節點，產出「系統混亂拓撲圖」。COO 負責向客戶 CTO 說明，在現有的混亂系統下，Agent 會如何失去靈魂，建立改變的急迫性。

### 階段 3：沙盒風格雕刻 (Sandbox Sculpting)

* **COO 行動：** 由創辦人（底層雕刻家角色）領軍，於我方 Vibe-Sandbox 環境中開始死磕模型。
* **營運控管：** COO 每週監控調校進度，控管「導航體力」的消耗，阻止團隊陷入永無止境的完美主義黑洞。確保 Temperature、Embedding 權重與 RAG 檢索邏輯在時程內收斂。

### 階段 4：極限施壓與邊界確認 (Stress Testing)

* **COO 行動：** 當沙盒風格成型後，把環境移交給 **【極限環境調校師 Stress Calibrator】**。
* **營運控管：** 啟動自動化施壓腳本。COO 密切關注 Quality Gate 的通過狀況。若 Agent 在高並發或惡意對話中出現流俗、跳針或瓦解，立刻退回階段 3 重新調校。

### 階段 5：法典建構與交付 (Codex Construction)

* **COO 行動：** 指派 **【轉譯佈道師 Blueprint Translator】** 進入專案。
* **營運控管：** 督促 Translator 將階段 3 的數學參數與階段 4 的抗壓測試結果，整合編纂為《品牌 AI 靈魂導航法典》，並部署線上「互動沙盒 Playground」。COO 親自進行交付物的最終視覺與文字質感校對（確保符合本公司風格架構師的審美標準）。

## E 區：交付物結構 (Deliverables Spec)

當專案結案、向客戶請款時，我們交付的不是冰冷的代碼庫，而是一套兼具體驗、數據與思維轉型的「終極包裹」：**【導航法典與互動沙盒 (The Codex & Interactive Sandbox)】**。

其結構必須嚴格包含以下兩大核心組件：

### 組件一：互動沙盒 Playground (The Interactive Sandbox)

這是一個由我方完全託管、部署於雲端的專屬線上測試網址。

* **功能：**
讓客戶的高層（執行長、產品總監）與工程團隊不需更改自己任何一行 Code，就能直接登入網址，親自動手與這個「完美調校、充滿靈魂與 Vibe」的黃金示範級 Agent (Golden Copy) 進行實時對話。
* **展示重點：**
* **動態 Vibe 切換器：** 網頁側邊欄展示不同的參數矩陣，讓客戶直觀感受 Temperature 與 Embedding 權重微調前後的巨大氣氛差異。
* **極限模擬開關：** 內建一個「模擬惡意攻擊/高延遲」按鈕，按下後系統會自動輸入攻擊詞，向客戶展示該 Agent 在極端逆境下依然保持品牌人設的優雅姿態。



### 組件二：品牌 AI 靈魂導航法典 (The Navigation Codex)

一份經過精美排版、具備高度專業感的純 Markdown/PDF 終極智庫文件，內容包含三大章節：

* **第一章：靈魂美學量化矩陣 (Vibe Blueprint)**
* **核心參數：** 詳列推薦的模型基礎設定值（如精準調校的 `Temperature` 變動曲線、`Top_P` 區間、`Frequency Penalty` 等）。
* **底層邏輯：** 說明知識庫向量檢索（Vector Search）的權重比例配方，以及 Embedding 維度過濾的邏輯，用數學和資工邏輯解釋「美感與氛圍」是如何被製造出來的。


* **第二章：抗壓與防崩韌性報告 (Resilience Log)**
* 詳列 【極限環境調校師】 所使用的百次攻擊測試腳本與對抗數據。
* 標明防範 Prompt Injection 的底層防禦護欄設定。


* **第三章：工程實作防呆手冊 (Implementation SOP)**
* 由 【轉譯佈道師】 撰寫的 API 串接指南與程式碼範例。
* 一步驟一步驟指導客戶端的 Junior 工程師，如何將沙盒中的參數，精確無誤地搬移到他們那堆混亂的現有架構中，且保證不失真。
